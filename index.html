ScoreSaber 2 is coming...

ScoreSaber 2 launches on Monday, May 25 at 4:00pm EST.

ScoreSaber has existed since May 8, 2018. When first made, the goal was simple: just a friendly neighbourhood leaderboard with support for custom songs. Then PPv1 / PPv2 happened, the ranking system happened, replay systems 1 & 2 happened.

That growth felt amazing and it was super fun to iterate so fast back then, but the foundation underneath didn't exactly keep up.

For years, new features were bolted onto systems that were never designed for them. Sometimes that worked... Sometimes it very much didn't. Eventually, almost every meaningful change had to fight the old backend, the old database, the old deployment setup or all 3 at once.

So what is ScoreSaber 2?

It's pretty much the update that gets moving again (and a bunch of other cool stuff, promised—just keep reading).

The Re-code

The old database went all the way back to the first version of ScoreSaber. Every player, leaderboard and score was sitting on top of decisions made when mostly just trying to get the thing working.

It held on for dear life for a surprisingly long time, but development slowly turned into archaeology. It was time for that to stop.

The database layer has moved to postgres with a completely redesigned data model. The backend has a new API, much stricter validation, proper background jobs, better monitoring and logging.

The frontend has also been rebuilt with Next.js around the new API, moving away from the very, very beta version of svelte-kit used on the old frontend which has kept us shackled for so long.

For third party devs, the new public API is properly automatically generated and has OpenAPI support.

Also—the old API isn't being switched off; ScoreSaber 2 has a compatibility layer for the existing website API, game API and legacy clients. New things will live on the new API.

Bus Factor

For a long time, because of how ScoreSaber was deployed, deep server access wasn't distributed widely. The setup worked, but it relied on old infrastructure, manual processes, and isolated context.

ScoreSaber 2 fixes a lot of that. Deployment is simpler, more automated and much easier for trusted developers to operate safely. Qwasyx will have full access to the new deployment platform for maintenance and custom jobs like reweights.

Once the dust settles, plans are in place to open up the ScoreSaber development team to more people and open source a much larger chunk of ScoreSaber.

Realms

Realms are one of the reasons the re-code was worth doing. The idea was originally inspired by ReBeat, a scoring system designed by Astrella.

A realm is basically a separate scoring context inside ScoreSaber. It can have its own ranked maps, PP curve, modifier values, player stats, history and leaderboards.

At launch, everyone will use the main realm, which is ScoreSaber as known today. Realms let new ranking systems and PP curves be tested on real scores without disrupting the main realm.

More Data

The old ScoreSaber data model was pretty lossy. In many places, only the latest state of something was stored, not how it arrived there. In ScoreSaber 2, every submitted score is recorded and kept indefinitely.

A lot more state changes are being logged as structured data now. Player aliases, country changes, bans, role changes, leaderboard status changes, PP changes, ranking request activity and replay views all have real structure behind them.

The next mod refresh will also allow the submission of fails and quits to help understand play patterns and map difficulties.

Accounts

Meta account handling has been one of the longest-running annoyances in ScoreSaber. Historically, Steam was the clean path for website login. If playing through the Meta Link app, accounts sat in a strange place.

ScoreSaber 2 allows website sign-in with a Meta account to access standard community features.

Replay Storage

The new system defines clear rules for replay retention boundaries:

  • Every current PB replay upload is stored first.
  • Every current PB replay is guaranteed to remain available for at least 30 days.
  • Ranked leaderboards keep the top 1,000 current PB replays indefinitely.
  • Unranked leaderboards keep the top 15 current PB replays indefinitely.
  • Replaced PB replays are kept for 7 days before cleanup runs.

The New Website

It's faster, cleaner, mobile-friendly and server-rendered.

Maps

The maps page is built around mapsets instead of a flat list of leaderboards. Difficulties are now grouped cleanly under the mapsets they belong to.

Rankings

Navigation allows jumping directly to specific positions, viewing friends, and filtering active/inactive status.

Leaderboards

Difficulties and game modes switch much more fluidly. Scores filter by friends, country, and proximity.

Scores

Scores carry deeper context, integrating explicit tracking for 58 headset labels and 40 controller profiles to map per-hand accuracy, cut quality, and hit streaks.

Social

ScoreSaber now features a follower and following system to track social connections across platforms.

Mods

The new backend maintains full backward compatibility, so current client mods continue functioning normally ahead of the mid-June refresh.

Project Ludus

Ludus allows high-quality, synchronized spectating and tournament broadcasting, rendering multiple viewpoints simultaneously with minimal overhead.

PPv3

Once ScoreSaber 2 stabilizes, PPv3 will enter active testing via isolated realms using real historical player data profiles.

Wrapping Up

This update represents a structural turning point enabling active, ongoing iteration. See you on the 25th.
